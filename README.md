# Drone-ID-Ocusync-3.0 

**DJI DroneID Demodulation Attempt (Mavic 3 Classic)**

This repository contains an implementation and adaptation of the DroneID demodulation techniques described by Schiller et al. (2023). The primary goal of this project was to capture and demodulate the DroneID signal from a DJI Mavic 3 Classic using a USRP B200 Software Defined Radio (SDR).

---
## Python Script - Paper Code: DroneID Analysis
This repository provides tools for the extraction, demodulation, and analysis of DJI DroneID.

### Directory structure:

- Paper/: Stores plots generated by the scripts and the analysis notebook.
- Cut_droneid/: Contains individual signal segments extracted from full recordings.
- Signal_analyzer.ipynb: A custom notebook developed for in-depth analysis of signal processing steps and visualization.
- .py Scripts: The core components used for demodulating and decoding signals.

### Usage: Offline Demodulation
To replicate the results found in our report, you can use either the Signal_analyzer.ipynb notebook or the droneid_receiver_offline.py script.

Using the Python script
> Run the following command in your terminal:
> Bash
>  python3 droneid_receiver_offline.py -i drone_id_cut19.51.iq --legacy -s 40e6
> Arguments:
> - -i : Input IQ file path.
> - --legacy : Enables legacy mode (as specified in the report).
> - -s : Sets the sample rate (e.g., 40e6 for 40MHz).

### Implementation & Modifications
The code in this repository is based on the original research artifacts. However, we have implemented several custom modifications to adapt the demodulator to our specific hardware setup and signal environment.

# Current Status and Known Issues

Despite the modifications, we have not yet achieved a successful full demodulation of the DroneID packets. While the signal is visible in the frequency domain, the extraction of the payload remains problematic.

Critical Challenges:
Constellation Plot: The most significant issue lies in the constellation visualization. Instead of clear, distinct clusters (BPSK/QPSK), the plot shows significant noise, rotation, or smearing, indicating issues with phase synchronization or frequency offset correction.
